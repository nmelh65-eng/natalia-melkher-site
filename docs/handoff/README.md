# Экспорт структуры и исходников для другой нейросети

Здесь лежат **инструкции** и скрипты. Сами тяжёлые файлы экспорта в репозиторий не коммитятся (см. `.gitignore`).

## 1. Сгенерировать полный Markdown

Из корня проекта:

```bash
node scripts/export-project-handoff.mjs
```

Появится файл **`docs/handoff/PROJECT_FULL_EXPORT.md`** (~350k+ символов): дерево всех файлов из `git ls-files` и **полное содержимое** текстовых файлов (без бинарников: картинки, шрифты и т.д.; `package-lock.json` не включается).

## 2. Получить PDF с кириллицей (рекомендуется)

Автоконвертеры в PDF часто ломают русский текст. Надёжный способ:

```bash
node scripts/export-markdown-to-print-html.mjs
```

Откройте **`docs/handoff/PROJECT_FULL_EXPORT_PRINT.html`** в **Chrome** или **Edge** → **Печать** → принтер **«Сохранить как PDF»** → сохраните файл.

Так вы получите один PDF со всей структурой и кодом в UTF-8.

## 3. Передача в Claude / другой модели

- Вариант A: загрузить получившийся **PDF**.
- Вариант B: загрузить **`PROJECT_FULL_EXPORT.md`** (если лимит контекста позволяет).
- Вариант C: разбить `.md` на части по папкам вручную или скриптом и отправлять по очереди.

## 4. Что не входит в экспорт

- Всё не из `git ls-files`
- `node_modules`, `.next`
- Бинарные пути перечислены в начале Markdown отдельным списком
- `package-lock.json` — см. `package.json` в экспорте
